# Design System: Vershkov

## 1. Overview

**Creative North Star: "Safe Sanctuary" ("Безопасное прибежище")**

Визуальный язык личного кабинета призван снизить тревожность, создать ощущение порядка, безопасности и поддержки. Он опирается на мягкую, естественную палитру с преобладанием нежного фиолетового и успокаивающего синего цветов, обилие свободного пространства ("воздуха") и плавные, скругленные формы. Элементы интерфейса предсказуемы, отзывчивы и не перегружают пользователя.

В отличие от публичного сайта, где крупная акцентная типографика (Dela Gothic) привлекает внимание, в личном кабинете дизайн служит функциональности. Шрифт Inter обеспечивает идеальную читаемость данных, а скругленные карточки организуют контент в логические блоки.

**Key Characteristics:**
- Спокойный и дружелюбный интерфейс с акцентом на контент.
- Мягкие, приглушенные оттенки и скругленные углы (12px).
- Четкая визуальная иерархия без декоративного шума.
- Идеальная адаптивность для мобильных устройств (крупные тач-зоны, сворачиваемый сайдбар).

## 2. Colors

Палитра личного кабинета построена по стратегии Restrained (сдержанная). Акцентный розовый и вспомогательный синий цвета используются строго для фокусировки внимания на ключевых действиях и состояниях, а нейтральные фоны обеспечивают спокойную рабочую среду.

### Primary
- **Lavender Purple** (`#B78EF6` / oklch(76% 0.16 264)): Основной брендовый цвет. Используется для главного акцента, активных состояний меню и основных кнопок. На темном фоне трансформируется в **Muted Violet** (`#a97bf3` / oklch(72% 0.15 264)) для лучшей контрастности.

### Secondary
- **Calm Blue** (`#4278a6` / oklch(52% 0.12 245)): Вспомогательный цвет, символизирующий спокойствие и ясность. Используется для информационных плашек, второстепенных индикаторов и элементов статистики.

### Neutral
- **Clean White** (`#ffffff`): Фон контента в светлой теме.
- **Deep Navy Grey** (`#14161d` / oklch(16% 0.02 220)): Фон контента в темной теме. Обеспечивает комфортное чтение при слабом освещении.
- **Warm Dark** (`#343434` / oklch(27% 0.00 0)): Основной цвет текста в светлой теме.
- **Warm Grey** (`#eceaea`): Границы и фоны неактивных элементов.

### Named Rules
**The 10% Accent Rule.** Яркий фиолетовый акцент не должен занимать более 10% площади любого экрана личного кабинета. Он служит ориентиром для взгляда и указывает на ключевое действие (записаться на сессию, пройти опрос).

---

## 3. Typography

В личном кабинете используется единая гарнитура **Inter** для обеспечения максимальной читаемости. Декоративные шрифты (Dela Gothic и Caveat/Sunless Day) используются исключительно на публичных страницах или в редких приветственных надписях, чтобы не создавать визуального шума при заполнении форм или чтении расписания.

**Body Font:** Inter (с fallback: `sans-serif`)

### Hierarchy
- **Title (h1)** (semibold, 24px/30px на мобильных, line-height 1.2): Заголовки страниц.
- **Headline (h2)** (semibold, 18px/20px, line-height 1.25): Заголовки разделов и карточек.
- **Body** (regular, 14px/16px, line-height 1.4): Основной текст, описание полей, инструкции.
- **Label / Small** (medium, 12px, letter-spacing 0.05em): Подписи к полям, мелкие детали, метаданные.

---

## 4. Elevation

Дизайн личного кабинета стремится к плоскостной эстетике с выраженным разделением слоев за счет цвета фона (Tonal Layering), а не глубоких теней. Это делает интерфейс современным и чистым.

### Shadow Vocabulary
- **Card Shadow** (`0 1px 3px rgba(0, 0, 0, 0.05), 0 1px 2px rgba(0, 0, 0, 0.03)`): Легкая, почти незаметная тень для карточек в светлой теме, помогающая отделить их от фона.
- **Interactive Hover** (`0 10px 15px -3px rgba(0, 0, 0, 0.05), 0 4px 6px -2px rgba(0, 0, 0, 0.03)`): Появляется только при наведении на интерактивные карточки-кнопки.

### Named Rules
**The Flat-By-Default Rule.** В неактивном состоянии все элементы лежат на одной плоскости. Легкая тень или изменение тона фона служат исключительно для обозначения интерактивности при наведении (hover) или фокусе.

## 5. Components

### Buttons
- **Shape:** Скругленные углы со средним радиусом (`12px` / `--radius`).
- **Primary:** Фоновый цвет Lavender Purple (`#B78EF6`), текст Warm Dark (`#343434`). На мобильных устройствах высота кнопки должна быть не менее 44px для удобства нажатия.
- **Hover / Focus:** Плавное изменение цвета фона на чуть более светлый фиолетовый (`#cbb0f9`), появление мягкого кольца фокуса (`--ring`).

### Cards / Containers
- **Corner Style:** Радиус скругления (`12px` / `--radius-m`).
- **Background:** Чистый белый в светлой теме, Deep Navy Grey (`#14161d`) в темной теме.
- **Border:** Тонкая граница (`1px`) цвета `--border` (или `--border-dark` в темной теме) для четкого разделения блоков.
- **Internal Padding:** `24px` на десктопах, адаптивно уменьшается до `16px` на мобильных экранах.

### Inputs / Fields
- **Style:** Мягкие скругленные углы (`8px`), тонкая рамка (`1px` `--border`), высота `44px` на мобильных.
- **Focus:** Фиолетовое фокус-кольцо (`--ring`) с небольшим размытием, подчеркивающее активное поле ввода.

### Navigation (Sidebar)
- **Style:** Боковая панель приглушенного оттенка (`--sidebar-background`) со сворачиваемой конструкцией.
- **Mobile treatment:** Полное скрытие сайдбара на экранах `< 768px` с вызовом через гамбургер-меню в шапке. Навигация должна быть адаптирована под жест смахивания (swipe).

## 6. Do's and Don'ts

### Do:
- **Do** Всегда проверяйте контрастность текста на кнопках (особенно на фиолетовом фоне `--primary`).
- **Do** Используйте скелетоны вместо спиннеров при загрузке данных в карточках статистики и расписания.
- **Do** Адаптируйте все таблицы и списки под мобильные экраны (трансформируйте таблицы в карточки на экранах `< 640px`).
- **Do** Делайте поля ввода высотой не менее 44px для удобного тапа пальцем на смартфонах.

### Don't:
- **Don't** Не используйте декоративные цветные полоски слева/справа у карточек (Side-stripe borders).
- **Don't** Не используйте градиентный текст (Gradient text) в личном кабинете.
- **Don't** Избегайте бесконтрольного наслоения карточек друг на друга (Nested cards).
- **Don't** Не используйте шрифты с засечками или Dela Gothic для отображения данных пользователей и подписей интерфейса.
- **Don't** Не блокируйте интерфейс долгими анимациями загрузки (любая анимация переходов должна быть ≤ 200мс).
