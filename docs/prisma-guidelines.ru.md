# 📘 Краткое руководство по Prisma

Это руководство содержит основные команды и правила для повседневной работы с базой данных и ORM Prisma в нашем проекте.

---

## 1. Базовый цикл разработки (Как добавлять новые таблицы/поля)

Ваша ежедневная работа с БД обычно состоит из 3 шагов:

### Шаг 1: Изменение схемы

Внесите нужные изменения (добавьте модель, поле, связь) в файл `prisma/schema.prisma`.

### Шаг 2: Создание и применение миграции

**Всегда** используйте эту команду после внесения изменений в `schema.prisma`:

```bash
npx prisma migrate dev --name имя_вашей_миграции
```

*(Например: `npx prisma migrate dev --name add_user_avatar`)*

**Что делает эта команда?**

1. Применяет ваши изменения к локальной/облачной базе данных (указанной в `.env`).
2. Создает SQL-файл в папке `prisma/migrations` для переноса этих изменений на другие сервера.
3. **Автоматически генерирует TypeScript типы** (обновляет локального клиента `PrismaClient`), чтобы редактор кода "
   увидел" ваши новые колонки и перестал выдавать ошибки типов.

### Шаг 3: Перезапуск сервера (если нужно)

Иногда Next.js может кэшировать серверные действия. Если типы обновились, но страница выдает ошибки, просто
перезапустите сервер `npm run dev`.

## 2. Полезные команды на каждый день

### Генерация типов без миграции

Если вы сделали `git pull` и скачали чужую миграцию, база данных уже может быть обновлена (или вы ее обновили), но ваш
локальный код ещё об этом "не знает" (в редакторе красные ошибки типов). Выполните:

```bash
npx prisma generate
```

Это обновит типы `PrismaClient` на основе текущего файла `schema.prisma`.

### Быстрые прототипы (БЕЗ миграций)

Иногда вы 50 раз за день меняете схему ради тестов и не хотите засорять папку `migrations` мелким мусором. Для этого
есть команда отправки схемы в базу напрямую:

```bash
npx prisma db push
```

> [!WARNING]
> Используйте `db push` с осторожностью в рабочих проектах, так как она не создает историю sql-миграций. Если вы
> добавили важную фичу, лучше используйте `migrate dev`.

## 3. Решение частых проблем и ошибок

### ❌ Ошибка "Invalid delegate.findMany() invocation"

**Причина:** Ваш код пытается запросить модель, которой ещё "нет" в локальном клиенте.
**Решение:** Выполните `npx prisma generate` для обновления типов или `npx prisma migrate dev`, если вы забыли применить
схему.

### ❌ Ошибка "Drift detected" (Рассинхрон базы и миграций)

**Причина:** Вы меняли таблицу прямо в базе (или через `db push`), но история файлов миграций не знает об этом
изменении.
**Решение (Если данные не жалко очистить):**

```bash
npx prisma migrate reset
```

*(База полностью очистится и пересоздастся заново на основе папки `migrations`)*

**Решение (Если данные нужно сохранить - Умный сброс истории / Baselining):**
Этот способ полезен, если история миграций сломалась, но в базе лежат важные данные (продакшн), и вы не можете выполнить сброс. Суть метода — стереть старую историю локально и создать новую базовую миграцию, "обманув" Prisma, что эта миграция уже применена на сервере.

1. Закомментируйте в `schema.prisma` любые **новые** изменения (оставьте только то, что *реально* уже есть в базе сейчас).
2. Удалите сломанную папку с историей:
   ```bash
   rm -rf prisma/migrations
   ```
3. Создайте новую папку для инициализирующей миграции:
   ```bash
   mkdir -p prisma/migrations/0_init
   ```
4. Сгенерируйте SQL-скрипт из вашей текущей схемы, ничего не записывая в БД:
   ```bash
   npx prisma migrate diff --from-empty --to-schema prisma/schema.prisma --script > prisma/migrations/0_init/migration.sql
   ```
5. Зарегистрируйте эту миграцию в БД как "уже выполненную", чтобы Prisma не пыталась пересоздать существующие таблицы:
   ```bash
   npx prisma migrate resolve --applied 0_init
   ```
6. Раскомментируйте ваши **новые** изменения в `schema.prisma`.
7. Запустите штатное создание новой миграции:
   ```bash
   npx prisma migrate dev --name <ваше_имя_миграции>
   ```

## 4. Визуальный просмотр базы данных

В Prisma встроен очень мощный аналог PhpMyAdmin, где вы можете смотреть таблицы, удалять и менять данные кликами мышки:

```bash
npx prisma studio
```

Он откроется в браузере по адресу `http://localhost:5555`.
