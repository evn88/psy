# 🧠 Vershkov.com

> **Профессиональная платформа для психолога** — современное веб-приложение для психолога Анны Вершковой, построенное на
> Next.js
> 16 с функциями управления клиентами, системой опросов и безопасной аутентификацией через несколько провайдеров.

🔗 **Сайт:** [vershkov.com](https://vershkov.com)

---

## ✨ Возможности

### 🔐 Аутентификация и безопасность

- **Многопровайдерная аутентификация** — Email/Password, Google OAuth и WebAuthn (Passkeys) через NextAuth v5
- **Проверка email** — Новые пользователи должны подтвердить свой email перед доступом к платформе
- **Приветственные уведомления** — Автоматические приветственные письма при регистрации через Google (не при повторном
  входе)
- **Ограничение попыток входа** — Дросселирование попыток входа (3 попытки в час)
- **Управление доступом по ролям** — Роли `ADMIN`, `USER` и `GUEST` с гранулярной защитой маршрутов

### 📋 Система опросов

- Опросы, созданные администратором, с различными типами вопросов (выбор одного, выбор нескольких, свободный текст,
  шкала)
- Назначение пользователям и отслеживание завершения
- Просмотр результатов с обсуждением комментариев между администратором и пользователями
- Автосохранение черновиков ответов на опросы

### 📧 Система email

- Интеграция **Resend** для транзакционных писем
- Массовая отправка из панели администратора (до 100 за раз)
- Отслеживание статуса доставки в реальном времени с опросом
- **Многоязычные шаблоны email** — письма отправляются на предпочитаемом языке пользователя
- Красивые шаблоны React Email (проверка, приветствие, сообщения администратора)

### 🌍 Интернационализация

- Полная поддержка **английского** и **русского** языков через `next-intl`
- Определение локали по cookie (без префиксов в URL)
- Предпочтение языка пользователя сохраняется в профиле
- Расширяемая — добавляйте новые языки, создавая файл переводов

### 🎨 UI и дизайн

- Поддержка тем Dark/Light/System через `next-themes`
- Адаптивный макет с дизайном, ориентированным на мобильные устройства
- Библиотека компонентов, построенная на примитивах **Radix UI**
- Всплывающие уведомления через **Sonner**
- Шрифт Inter с поддержкой латиницы и кириллицы

### 👤 Личный кабинет пользователя

- Личный кабинет со статистикой
- Управление опросами (в ожидании/завершённые)
- Управление профилем (имя, привязка Google-аккаунта)
- Управление passkey (создание/удаление)
- Настройки внешнего вида и языка

### 🛠 Панель администратора

- Управление пользователями (CRUD, назначение ролей)
- Создание опросов и назначение их пользователям
- Инструмент для массовой рассылки писем
- Отслеживание статуса пользователей в реальном времени (heartbeat)

## 🚀 Начало работы

### Требования

- Node.js 18+
- База данных PostgreSQL
- API ключ [Resend](https://resend.com)
- Учётные данные Google OAuth (опционально)

### Установка

```bash
# Клонируем репозиторий
git clone https://github.com/evn88/psy.git
cd psy

# Устанавливаем зависимости
npm install

# Настраиваем переменные окружения
cp .env.example .env.local

# Генерируем клиент Prisma
npx prisma generate

# Запускаем миграции БД
npx prisma migrate deploy

# Запускаем сервер разработки
npm run dev
```

### Переменные окружения

#### Аутентификация и база данных

| Переменная            | Описание                                                                                          |
|-----------------------|---------------------------------------------------------------------------------------------------|
| `AUTH_SECRET`         | Секретный ключ NextAuth для подписи сессий (генерировать: `openssl rand -base64 32`)              |
| `AUTH_GOOGLE_ID`      | ID клиента Google OAuth (ранее `GOOGLE_CLIENT_ID`)                                                |
| `AUTH_GOOGLE_SECRET`  | Секрет клиента Google OAuth (ранее `GOOGLE_CLIENT_SECRET`)                                        |
| `DATABASE_URL`        | Строка подключения PostgreSQL (полный URL с учётными данными)                                     |
| `POSTGRES_URL`        | Альтернативная строка подключения Postgres (используется Vercel Postgres)                         |
| `PRISMA_DATABASE_URL` | Явный URL базы данных Prisma (переопределяет `DATABASE_URL` если установлена)                     |
| `ADMIN_EMAIL`         | Email-адрес для предоставления доступа администратора при первом входе (например, your@email.com) |

#### Email и уведомления

| Переменная                              | Описание                                                                            |
|-----------------------------------------|-------------------------------------------------------------------------------------|
| `RESEND_API_KEY`                        | API ключ Resend для транзакционных писем                                            |
| `VAPID_PRIVATE_KEY`                     | Приватный ключ VAPID для push-уведомлений (держать в секрете)                       |
| `NEXT_PUBLIC_VAPID_PUBLIC_KEY`          | Публичный ключ VAPID для push-уведомлений (безопасно открыть)                       |
| `VAPID_SUBJECT`                         | VAPID subject URI (например, `mailto:your@email.com`) для идентификации уведомлений |
| `CRON_SECRET`                           | Секрет для защиты ежедневного Vercel Cron эндпоинта `/api/cron/session-reminders`   |
| `WORKFLOW_MONTHLY_STEP_LIMIT`           | Опциональный месячный бюджет Workflow Steps для алертов (по умолчанию: `50000`)     |
| `WORKFLOW_ALERT_THRESHOLD_PERCENT`      | Опциональный порог предупреждения в процентах (по умолчанию: `80`)                  |
| `WORKFLOW_ESTIMATED_STEPS_PER_REMINDER` | Опциональная оценка steps на один запуск workflow-напоминания (по умолчанию: `3`)   |

#### PayPal платежи

| Переменная                      | Описание                                                                                         |
|---------------------------------|--------------------------------------------------------------------------------------------------|
| `NEXT_PUBLIC_PAYPAL_CLIENT_ID`  | Client ID PayPal REST приложения для JavaScript SDK и серверных вызовов Orders API               |

#### URL приложения

| Переменная            | Описание                                                                                           |
|-----------------------|----------------------------------------------------------------------------------------------------|

### Настройка администратора

1. **Зарегистрируйтесь** — Создайте учётную запись с вашим email или через Google OAuth
2. **Установите ADMIN_EMAIL** — Добавьте ваш email в `.env.local`:
   ```bash
   ADMIN_EMAIL=your@email.com
   ```
3. **Войдите снова** — Войдите в систему снова, и ваша учётная запись автоматически будет повышена до роли `ADMIN`
4. **Перейдите в панель администратора** — Откройте `/admin` (доступно только пользователям с ролью ADMIN)

## 📱 PWA: офлайн-режим

Приложение работает офлайн благодаря Service Worker (`public/sw.js`).

### Какие страницы доступны офлайн

| Страница          | Путь           |
|-------------------|----------------|
| Главная           | `/`            |
| Личный кабинет    | `/my`          |
| Расписание сессий | `/my/sessions` |
| Опросы            | `/my/surveys`  |

### Как добавить новую страницу в офлайн-кеш

1. Откройте `public/sw.js`
2. Найдите массив `PRE_CACHED_URLS`
3. Добавьте нужный путь:

```js
const PRE_CACHED_URLS = [
    '/',
    '/my',
    '/my/sessions',
    '/my/surveys',
    '/my/ваша-страница',  // ← добавить сюда
];
```

4. Задеплойте. После обновления SW новые страницы появятся в кеше автоматически.

> Чтобы сбросить старый кеш у всех пользователей (например после крупного обновления) — измените константу:
> ```js
> const CACHE_VERSION = 'v2'; // было v1
> ```

### Стратегии кеширования

| Тип запроса        | Стратегия                                 |
|--------------------|-------------------------------------------|
| `/api/**`          | Network Only — никогда не кешировать      |
| `/_next/static/**` | Cache First — иммутабельные JS/CSS бандлы |
| PNG/SVG/ICO        | Cache First — статические иконки          |
| HTML-страницы      | Network First → fallback кеш              |

## 🔔 Push-уведомления

### Первоначальная настройка (один раз)

1. Сгенерировать VAPID-ключи:
   ```bash
   npx web-push generate-vapid-keys
   ```

2. Добавить в `.env.local` и на сервере:
   ```env
   NEXT_PUBLIC_VAPID_PUBLIC_KEY=<публичный ключ>
   VAPID_PRIVATE_KEY=<приватный ключ>
   VAPID_SUBJECT=mailto:admin@vershkov.com
   ```

### Как работает

- При первом визите пользователю показывается баннер с запросом разрешения
- Подписка хранится в таблице `PushSubscription` в БД
- Управление: `/my/settings` → «Push-уведомления»
- Если пользователь заблокировал — показываются инструкции по разблокировке

### Поддержка платформ

| Платформа                        | Статус                  |
|----------------------------------|-------------------------|
| Android Chrome/Firefox           | ✅ Полная                |
| iOS Safari 16.4+ (PWA на экране) | ✅ Работает              |
| iOS Safari (браузер)             | ❌ Apple не поддерживает |
| Desktop Chrome/Firefox/Edge      | ✅ Полная                |

### Отправка из админки

1. `/admin/send-email` → выбрать получателей → написать сообщение (до 178 символов)
2. Кнопка **«Отправить Push»** → подтвердить
